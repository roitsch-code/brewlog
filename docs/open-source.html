Fork it

Personalised by design.
Open by default.

BrewLog is deeply opinionated — built around one person's equipment, grinder, water setup, and taste profile. But it was built to be forked.

The stack

Deliberately
minimal.

No exotic dependencies. No proprietary lock-in. Everything is replaceable.

  • Framework Next.js 14 App Router + TypeScript
  • Database Firebase Firestore (your own instance)
  • Storage Firebase Storage (bag photos)
  • AI Claude API — Sonnet 4.6 for recommendations, Haiku for lightweight tasks
  • Hosting Vercel (one-click deploy)
  • Auth WebAuthn / Face ID + PIN fallback, JWT sessions
  • Platform iPhone PWA — add to home screen, runs like a native app
→ Your brew history never leaves your own Firestore database. Claude API calls are stateless — no session data persists outside your instance. Face ID keeps the app private on device.
What's personal

Two files hold
everything opinionated.

The entire personality of the recommendation engine lives in two places. Swap those out for your own setup and BrewLog becomes your tool, not someone else's.

src/lib/claude/recommend.ts

The brew recommendation system prompt. Contains the full equipment list (every brewer with capacity limits), grinder reference (Niche° settings or Comandante clicks), water setup, taste baseline, championship recipes, timing rules, and the novelty logic for experiment mode. This is where your setup lives.

src/app/api/match/route.ts

The coffee match scoring logic. Analyses your brew history for taste patterns — which origins, processes, and methods correlate with your highest ratings — and uses that to evaluate new coffees. The craft/fit attribution data feeds back here to separate execution quality from style alignment.

Getting started

Adapt it to your
setup in an afternoon.

1
Fork the repo and clone it

Everything is in github.com/roitsch-code/brewlog. The README has the full setup guide including Firebase project creation and environment variables.

2
Set your environment variables

Set USER_DISPLAY_NAME and USER_LOCATION in .env.local. Add your Anthropic API key, Firebase credentials, and auth secrets. The README lists every variable.

3
Update the equipment and taste profile

Edit recommend.ts with your brewers, grinder, water setup, and taste baseline. The system prompt is written in plain English — update the relevant sections for your kit. The CLAUDE.md file explains each section.

4
Deploy to Vercel

Push to GitHub, connect to Vercel, add the environment variables in the Vercel dashboard. One important note: set all env vars with printf 'value' | vercel env add KEY production — not with heredoc or echo, which add a trailing newline that will break WebAuthn.

5
Add to your iPhone home screen

Open the deployed URL in Safari, tap Share → Add to Home Screen. From that point it runs as a standalone PWA — full screen, no browser chrome, Face ID for auth.

One honest note

It is a personal tool,
not a product.

There is no issue tracker, no support, no roadmap. The code is annotated and the CLAUDE.md explains the architecture. If you brew seriously and want something that learns from your data — it should work. If something is broken for your setup, the codebase is small enough to debug yourself.
